Frequently Asked Questions

What is Exomiser?

Exomiser is a Java program that finds potential disease-causing variants from whole-exome or whole-genome sequencing data.

What file formats does Exomiser support?

Exomiser supports VCF files for variant data and uses HPO (Human Phenotype Ontology) terms for phenotype analysis.

How long does analysis take?

Analysis time varies depending on the size of your VCF file and the complexity of the analysis. Most analyses complete within 30 minutes.

What are HPO terms?

HPO (Human Phenotype Ontology) terms are standardized descriptions of phenotypic abnormalities encountered in human disease. They help Exomiser prioritize variants based on clinical phenotypes.

How do I create an individual record?

To create an individual record, navigate to the Individuals section and click "Add Individual". You'll need to provide basic information (name, age, sex), upload a VCF file, and add relevant HPO phenotype terms that describe the clinical features.

What genome assemblies are supported?

The application supports both hg19 (GRCh37) and hg38 (GRCh38) genome assemblies. Make sure your VCF file coordinates match the selected genome assembly for accurate analysis.

How do I start an analysis?

Once you have created an individual record with a VCF file and HPO terms, go to the Analysis section and click "New Analysis". Select the individual, configure analysis parameters (frequency threshold, pathogenicity threshold), and start the analysis.

Can I monitor analysis progress in real-time?

Yes! The application provides real-time progress monitoring. You can see the analysis status (PENDING, RUNNING, COMPLETED, FAILED) and view live output logs during execution. The progress is updated automatically without refreshing the page.

What analysis modes are available?

The default analysis mode is PASS_ONLY, which analyzes only variants that passed quality filters in your VCF file. You can also configure frequency thresholds (maximum allele frequency) and pathogenicity thresholds (minimum pathogenicity score) to customize the analysis.

How do I view analysis results?

Once analysis is completed, you can view the results by clicking on the analysis in your dashboard. Exomiser generates a comprehensive HTML report with prioritized variants, gene rankings, and detailed annotations that you can download or view directly in the browser.

What should I do if my analysis fails?

If an analysis fails, check the error message and log output for details. Common issues include invalid VCF format, incompatible genome assembly, or insufficient system resources. Ensure your VCF file is properly formatted and matches the selected genome assembly.

Can I edit individual records after creation?

Yes, you can edit individual records to update information such as medical history, diagnosis, or add/remove HPO terms. However, the VCF file cannot be changed once uploaded. If you need to use a different VCF file, create a new individual record.

How much memory does Exomiser require?

Exomiser is configured to use up to 4GB of memory (Java heap space). Large VCF files or whole-genome analyses may require more resources and take longer to complete. The system includes memory monitoring to track resource usage during analysis.

What is a phenopacket?

A phenopacket is a standardized format for sharing phenotypic and clinical data. The application automatically generates phenopacket YAML files from your individual records, containing the subject information, HPO terms, and metadata that Exomiser uses for analysis.

Is my data secure and private?

Yes, the application includes user authentication and authorization. Only registered users can access the system, and each user can only view and manage their own individual records and analyses. All uploaded files and generated results are stored securely with proper access controls.
